## Why this release matters

From **v1.0.0 → v1.0.6** this fork went from “pricedb fork with a version number” to a full **blank-bot starter kit** you can clone, configure, and run with the GUI panel — without leaking Steam secrets to GitHub.

**v1.0.6** is the first release that stacks *everything* we shipped so far on top of upstream **5.17.0**:

- Fresh public clone (no `.env`, no `files/`)
- Discord Autokeys + Unhalt + slash listing controls

- Safe `!updaterepo` (keeps `dist/` until build succeeds)
- Steam HTTP 429 / rate-limit resilience
- PriceDB Store + Mannco.store integrations from the 5.17 line
- Wiki + docs for setup

If you are starting today: **use this tag**. If you are upgrading: read the update section at the bottom.

## Fresh install tutorial (blank bot)

**Goal:** a clean bot + panel with *your* credentials only on *your* machine.

### 0. Requirements

- **Node.js 22+**
- Steam Guard Mobile Authenticator
- backpack.tf API key + access token
- [pricedb.io](https://pricedb.io) account (default pricer)
- Optional: Discord bot token, PriceDB Store key, Mannco API key, PM2

### 1. Clone the blank bot

```bash
git clone https://github.com/uwu6967/tf2autobot-pricedb.git
cd tf2autobot-pricedb
git checkout v1.0.6   # or stay on master after this release
npm install
npm run build
```

### 2. Create your secret `.env` (never commit this)

```bash
cp template.env .env
# or: cp .env.example .env
```

Fill at least:

```env
STEAM_ACCOUNT_NAME=...
STEAM_PASSWORD=...
STEAM_SHARED_SECRET=...
STEAM_IDENTITY_SECRET=...
STEAM_API_KEY=...
BPTF_ACCESS_TOKEN=...
BPTF_API_KEY=...
ADMINS=[{ "steam": "<your steamid64>", "discord": null }]
IPC=true
```

`files/` is gitignored — the bot creates per-account folders there at runtime (`options.json`, `pricelist.json`, polldata, etc.). That is intentional so GitHub stays a **blank** template.

### 3. Clone and start the GUI panel (v3.6.2)

```bash
git clone https://github.com/uwu6967/tf2autobot-gui-panel.git
cd tf2autobot-gui-panel
git checkout v3.6.2
npm install
cp template.env .env
npm run build
npm start
```

Open **http://localhost:3000**. The panel talks to the bot over IPC — it does **not** log into Steam itself.

Full panel walkthrough: [TUTORIAL.md](https://github.com/uwu6967/tf2autobot-gui-panel/blob/main/TUTORIAL.md)

### 4. Start the bot

```bash
cd tf2autobot-pricedb
node dist/app.js
# or: pm2 start dist/app.js --name tf2autobot --cwd /path/to/tf2autobot-pricedb
```

When both are up: pricelist, trades, settings, Unlisted Stock, and partial autoprice all flow through the panel.

## Feature tour — everything baked into v1.0.6

### 🔥 Upstream 5.17.0 RC

Merged the PriceDB upstream release candidate while **keeping** our Discord / Autokeys / partial-autoprice / listing UX work. You get Mannco.store commands, trade-URL hardening, dependency bumps, and Node 22 support without losing the fork features.

### 🤖 Discord Autokeys + Unhalt

Control Autokeys from Discord and unhalt the bot at startup without digging through Steam chat. Version-update Discord notifications from earlier 1.0.x releases are still in the tree.

### 💱 Partial autoprice (sell-only / buy-only)

Mix live pricedb prices with manual sides:

- Autoprice **sell only** while you set buy manually (or the reverse)
- Panel **v3.6.2** price modal exposes the toggles
- Bot understands `autoprice` + `isPartialPriced` (and related entry fields) over IPC and Discord slash listing controls

### 🖥️ Panel IPC that actually tells you the truth

From **v1.0.5** (included here): `addItem` / `updateItem` / `removeItem` emit **string** errors instead of opaque Error objects — so the GUI can show *why* a list failed.

Unlisted Stock `getInventory` IPC (from **v5.16.10 / v1.0.x** lineage) is included for the panel Unlisted page.

### 🛡️ Steam rate limits & safe updates

Carried forward from **v1.0.3–v1.0.4**:

- HTTP **429** escrow / trade-URL / localization failures no longer crash-loop PM2
- `!updaterepo` keeps a working `dist/` until `npm run build` succeeds
- Compile failures fail cleanly instead of wiping your running build

### 🏪 PriceDB Store + Mannco.store

- Mirror backpack.tf sell listings to your pricedb.io / crit.tf store (`%pricedb_store%` listing notes)
- Optional Mannco.store deposits, listings, buy orders, withdrawals (`!mcosell`, `!mcowithdraw`, `!mcostatus`, …)

### 📚 Docs & blank-bot hygiene

- Wiki / docs site under [uwu6967/tf2autobot-pricedb](https://github.com/uwu6967/tf2autobot-pricedb/wiki)
- `.env.example` + `template.env` for empty credentials
- README points at **uwu6967** bot + panel releases

## Upgrading from an older bot

### From v1.0.5 (or any 1.0.x)

```bash
cd /path/to/tf2autobot-pricedb
git fetch --tags
git checkout v1.0.6   # or: git pull origin master
npm install
npm run build
pm2 restart <your-bot>
```

Or from Steam chat (PM2 + git clone):

```
!updaterepo
```

Also bump the panel:

```bash
cd /path/to/tf2autobot-gui-panel
git fetch --tags
git checkout v3.6.2
npm install
npm run build
pm2 restart <your-panel>
```

### Checklist after upgrade

1. Confirm `.env` still has `IPC=true`
2. Spot-check Discord Autokeys / Unhalt if you use them
3. In the panel price modal, try sell-only / buy-only autoprice once
4. List or update an item and confirm error messages are readable if something fails
5. Run `!uptime` / check Discord version webhook if enabled

### From upstream TF2Autobot / PriceDB only

Treat this like a **new blank install**: clone `uwu6967/tf2autobot-pricedb` @ **v1.0.6**, copy your old `.env` + `files/<account>/` over carefully, then pair with panel **v3.6.2**. Do not commit those folders.
