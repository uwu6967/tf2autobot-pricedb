# v1.0.10 — FIFO purchase history & smarter sell reprice

Follow-up focused on **cost basis** after v1.0.7/v1.0.9.

**Pair with:** [tf2autobot-gui-panel v3.6.2](https://github.com/uwu6967/tf2autobot-gui-panel/releases/tag/v3.6.2)

---

## What's new

### 🧾 Purchase history on `!get` / Discord

- Shows **stock-only** FIFO lots still in inventory (sold lots drop off)
- Format: `46 items → 3 keys (56ref)` — paid price + key rate at buy time
- Discord sends a second embed (split across messages if long)
- Steam chat gets the same history as plain text

### 💱 Lot-accurate profit

- Accepted-trade profit uses each sold lot’s **actual paid** cost and that lot’s key rate when known
- New buys store `keyPriceMetal` on the FIFO entry for later history

### 📉 Sell reprice from next FIFO lot

After a sale removes the oldest lot(s), non-full-autoprice entries can set **sell** from the next oldest lot’s paid cost + `minProfitScrap` (PPU).

Skips keys/metal, full `autoprice`, and `autopriceSell` live sides.

### Discord embeds

- `sendAnswerEmbeds` can send multiple embeds (get + purchase history)
